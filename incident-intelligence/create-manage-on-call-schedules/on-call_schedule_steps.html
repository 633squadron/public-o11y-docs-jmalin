Given this example schedule scenario, follow these steps to create a schedule:

  1. In Incident Intelligence, go to Incident Response Configuration.
  2. Select the On-call schedules tab.
  3. Select Create new schedule
  4. Select the pencil icon to edit the schedule name. Schedule names should include a team name (so the schedule is easy to find when including in your incident workflows. The schedule names also often refer to priority (for example, primary, secondary, and so on), or what is being covered (for example, database, web, and so on). Select the checkmark icon to save your schedule name.
  5. Under Add shifts to this schedule, create one day-by-day shift and one week-by-week shift. There is a default blank shift to get you started in new schedules. To add shifts to the schedule, select Add new shift.
    1. Enter a shift name. Shift names often refer to time (for example, business hours, weekends, and so on).
    2. Select a shift start date.
    3. Use the timezone picker to select the timezone for the coverage.

    4. Define your shift coverage:

      Shift coverage type

      Steps

      For your Monday through Friday coverage, select Day-by-day shift coverage.
      1. Select the days you want to set up coverage for. For your scenario, select Monday through Friday.
      2. Select 9 AM for the Daily start time and 5 PM for the Daily end time. If you want to set up 24-hour coverage, select Create 24 hr shifts.
      3. Under Customize shift handoff, select 5. This is the number of shifts that you want to elapse before a handoff to the next team member in the schedule list. This selection configures the shift so that the same person is on call all week.
      4. Under Add team members, add one or more team members to the shift.
        1. Enter team member names and select them when they appear. The team member list comes from the Observability Cloud user list. See Create and manage users in Splunk Observability Cloud for more info.
        2. Drag team members in the list to reorder.
      5. Select Save shift. Shifts are added to the SAVED SHIFTS list under the schedule name.

      For your weekend coverage, select Week-by-week shift coverage.
      1. Select Friday and 5 PM in the Starts on drop-down lists.
      2. Select Monday and 9 AM in the Ends on drop-down lists.
      3. Under Customize shift handoff, select 1. This is the number of shifts that you want to elapse before a handoff to the next team member in the schedule list. This selection configures the shift so that the same person is on call for the entire weekend or one shift.
      4. Under Add team members, add one or more team members to the shift.
        1. Enter team member names and select them when they appear. The team member list comes from the Observability Cloud user list. See Create and manage users in Splunk Observability Cloud for more info.
        2. Drag team members in the list to reorder.
      5. Select Save shift. Shifts are added to the SAVED SHIFTS list under the schedule name.
